## Крок 5: Ручне налаштування на Render

### 5.1. Створити Web Service

1. Увійдіть на [dashboard.render.com](https://dashboard.render.com)
2. Натисніть **"New +"** → **"Web Service"**
3. Оберіть **"Connect a repository"**

### 5.2. Підключити репозиторій

1. Знайдіть свій репозиторій `sys-plus-bot`
2. Натисніть **"Connect Repository"**

### 5.3. Налаштувати сервіс

Заповніть поля:

```
Name: sys-plus-bot
Region: Frankfurt, Germany
Branch: main
Root Directory: (залиште порожнім)
Runtime: Python 3
Build Command: pip install -r requirements.txt
Start Command: python app.py
Instance Type: Free
```

### 5.4. Додати змінні оточення

Натисніть **"Advanced"** → **"Add Environment Variable"**:

| Key | Value |
|-----|-------|
| `TELEGRAM_BOT_TOKEN` | `8261578455:AAHP9G7DnwWVhNa-H3FLr5ZmMRfh367Df7A` |
| `KEYCRM_API_KEY` | `NGZmYzg4YjY2MTE0OGRkNTBmYWM1NmFjZjE5YWI3MDEwMzc4YmNjZA` |
| `KEYCRM_BASE_URL` | `https://keycrm.app/api/v2` |
| `TELEGRAM_ADMIN_IDS` | `ВАШ_ID_З_КРОКУ_1` |
| `PORT` | `5000` |

**Важливо:** Не додавайте `RENDER_EXTERNAL_URL` - Render сам її створить!

### 5.5. Запуск

1. Натисніть **"Create Web Service"**
2. Зачекайте 2-5 хвилин
3. Коли статус буде **"Live"** - готово!

## Крок 7: KeyCRM Webhook

### 7.1. Отримати Webhook URL

Ваш URL:
```
https://sys-plus-bot.onrender.com/webhook/keycrm
```

### 7.2. Налаштувати в KeyCRM

1. Увійдіть в [KeyCRM](https://keycrm.app)
2. ⚙️ Налаштування → Інтеграції → Webhooks
3. Додати webhook
4. Заповніть:

```
Назва: Sys+ Telegram Bot
URL: https://sys-plus-bot.onrender.com/webhook/keycrm
Метод: POST
Події: order.created, order.status_changed, client.created
```

### 7.3. Протестувати

1. Створіть тестове замовлення в KeyCRM
2. Перевірте Telegram
3. Бот має надіслати сповіщення!

## 🐛 Вирішення проблем

### ❌ Бот не відповідає

**Причина:** Сервіс "заснув" на безкоштовному тарифі

**Рішення:**
1. Відкрийте `https://sys-plus-bot.onrender.com/health`
2. Зачекайте 10-30 секунд
3. Спробуйте знову написати боту

### ❌ Помилка "Unauthorized"

**Причина:** Неправильний API ключ

**Рішення:**
1. Перевірте `KEYCRM_API_KEY` в Render
2. Переконайтеся що ключ дійсний

### ❌ Webhook не працює

**Причина:** Неправильний URL

**Рішення:**
1. Перевірте що URL починається з `https://`
2. Перевірте логи на Render
3. Переконайтеся що сервіс активний

### ❌ Git помилки

**Причина:** Проблеми з Git

**Рішення:**
1. Перевстановіть Git
2. Використайте GitHub Desktop
3. Завантажте файли через веб-інтерфейс GitHub

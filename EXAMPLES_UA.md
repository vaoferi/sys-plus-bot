## Автоматичні сповіщення

Бот автоматично надсилає сповіщення адмінам у таких випадках:

### ✅ Нове замовлення
- Миттєве сповіщення після створення замовлення в KeyCRM
- Містить ID, суму, ім'я клієнта
- Посилання на перегляд замовлення

### ✅ Зміна статусу
- Сповіщення при зміні статусу замовлення
- Показує новий статус
- Допомагає відстежувати виконання

### ✅ Новий клієнт
- Сповіщення про реєстрацію нового клієнта
- Контактна інформація
- Корисно для відділу продажів

## Форматування повідомлень

Бот використовує **HTML розмітку** Telegram:

- `<b>` ... `</b>` - **жирний текст**
- `<i>` ... `</i>` - *курсив*
- Емодзі для кращої візуалізації
- Чітка структура з розділами

## Безпека

### Хто отримує сповіщення?

Тільки адміністратори з `TELEGRAM_ADMIN_IDS`:
```
TELEGRAM_ADMIN_IDS=123456789,987654321
```

### Що якщо ID не вказано?

Бот працюватиме, але сповіщення нікому не надсилатимуться.

### Як дізнатися свій ID?

1. Відкрийте @userinfobot в Telegram
2. Натисніть `/start`
3. Скопіюйте числове значення

---

## Помилки та обробка

### Якщо KeyCRM недоступний

```
❌ Помилка отримання замовлень
```

### Якщо Telegram не доставляє повідомлення

Бот логує помилку, але продовжує працювати:
```
Failed to send message to admin 123456789: bot was blocked by the user
```

### Якщо webhook не налаштовано

Перевірте логи на Render - будуть помилки підключення.

## Тестування команд

### Локальне тестування

1. Запустіть бота локально:
```bash
python app.py
```

2. Використайте test_local.py для ngrok:
```bash
python test_local.py
```

3. Напишіть боту в Telegram

### Тестування на Render

1. Відкрийте health check URL
2. Зачекайте поки сервіс "прокинеться"
3. Надішліть команду боту

## Продуктивність

### Час відповіді

- **Локально:** < 1 секунди
- **Render (активний):** 1-2 секунди
- **Render ("спить"):** 10-30 секунд (перший запит)

### Обмеження

- Telegram API: ~30 повідомлень/секунду
- KeyCRM API: залежить від тарифу
- Render Free: 512 MB RAM

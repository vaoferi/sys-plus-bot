# 🔄 Інструкція з налаштування KeyCRM Webhook

## 📋 Передумови

- ✅ Бот вже розгорнуто на Render.com
- ✅ Ви знаєте URL свого бота (наприклад: `https://sys-plus-bot.onrender.com`)

---

## 🎯 Крок 1: Отримати Webhook URL

Ваш бот має спеціальний endpoint для отримання сповіщень від KeyCRM:

```
https://sys-plus-bot.onrender.com/webhook/keycrm
```

**Замініть** `sys-plus-bot` на назву вашого сервісу!

### Приклади:

| Назва сервісу | Webhook URL |
|---------------|-------------|
| `sys-plus-bot` | `https://sys-plus-bot.onrender.com/webhook/keycrm` |
| `my-crm-bot` | `https://my-crm-bot.onrender.com/webhook/keycrm` |
| `telegram-crm` | `https://telegram-crm.onrender.com/webhook/keycrm` |

## 🎯 Крок 3: Знайти розділ Webhooks

### Варіант A: Через меню

1. Натисніть на **⚙️ Налаштування** (ліве меню)
2. Оберіть **"Інтеграції"** або **"API"**
3. Знайдіть розділ **"Webhooks"** або **"Сповіщення"**

### Варіант B: Через API налаштування

1. Натисніть на своє ім'я (правий верхній кут)
2. Оберіть **"Налаштування"**
3. Перейдіть у вкладку **"Розробникам"** або **"API"**

---

## 🎯 Крок 4: Додати новий Webhook

Натисніть кнопку **"Додати webhook"** або **"Створити"**

### Заповніть поля:

| Поле | Значення |
|------|----------|
| **Назва** | `Sys+ Telegram Bot` |
| **URL** | `https://sys-plus-bot.onrender.com/webhook/keycrm` |
| **Метод** | `POST` |
| **Content-Type** | `application/json` |
| **Активний** | ✅ Так |

---

## 🎯 Крок 5: Обрати події

Оберіть події, на які бот має реагувати:

### ✅ Рекомендовані події:

#### Замовлення:
- ☑️ **order.created** - Створено нове замовлення
- ☑️ **order.status_changed** - Змінено статус замовлення
- ☑️ **order.deleted** - Видалено замовлення
- ☑️ **order.paid** - Замовлення оплачено

#### Клієнти:
- ☑️ **client.created** - Додано нового клієнта
- ☑️ **client.updated** - Оновлено дані клієнта

#### Товари:
- ☐ **product.created** - Додано товар (за бажанням)
- ☐ **product.updated** - Оновлено товар (за бажанням)

### 💡 Порада:
Обирайте тільки необхідні події, щоб не перевантажувати бота сповіщеннями.

## 🎯 Крок 6: Додаткові налаштування (опціонально)

### Фільтрація:

Якщо потрібно отримувати сповіщення тільки про певні замовлення:

```json

  }
}
```

### Headers:

За потреби додайте кастомні заголовки:

```

```

### Secret Key:

Для перевірки автентичності вебхуків (якщо підтримується):

```

```

| `order.created` | Створено замовлення | При додаванні нового замовлення |

## 🛠️ Верифікація вебхуків (Advanced)

### Як перевірити що вебхук прийшов саме від KeyCRM?

Бот може перевіряти сигнатуру в заголовках:

```python
from flask import request
import hmac
import hashlib

def verify_keycrm_signature(secret_key: str) -> bool:
    signature = request.headers.get('X-KeyCRM-Signature')
    if not signature:
        return False
    
    payload = request.get_data()
    expected_signature = hmac.new(
        secret_key.encode(),
        payload,
        hashlib.sha256
    ).hexdigest()
    
    return hmac.compare_digest(signature, expected_signature)
```

### Додайте secret key у налаштування вебхука:

1. Створіть випадковий рядок (наприклад, через [randomkeygen.com](https://randomkeygen.com))
2. Додайте його в налаштування вебхука в KeyCRM
3. Використовуйте для перевірки в коді бота

## 📊 Моніторинг вебхуків

### Перегляд історії відправок

У деяких CRM можна переглянути історію відправок вебхуків:

1. Налаштування → Webhooks
2. Оберіть свій webhook
3. Натисніть **"Історія"** або **"Логи"**
4. Перегляньте статуси відправок

### Що робити якщо webhook не працює?

#### ❌ Помилка "Failed to deliver"

**Причина:** URL недоступний

**Рішення:**
1. Перевірте чи сервіс на Render активний
2. Відкрийте `https://sys-plus-bot.onrender.com/health`
3. Зачекайте поки сервіс "прокинеться"
4. Спробуйте надіслати тестове сповіщення знову

#### ❌ Помилка "Invalid response"

**Причина:** Бот повертає неправильну відповідь

**Рішення:**
1. Перевірте логи на Render
2. Переконайтеся, що endpoint повертає `{'ok': True}`
3. Код обробника має бути у `app.py`

#### ❌ Webhook відключено

**Причина:** Багато невдалих спроб

**Рішення:**
1. Увімкніть webhook назад в налаштуваннях
2. Виправте помилку в коді бота
3. Протестуйте локально перед повторним увімкненням

Бот обробляє ~30 запитів/секунду (обмеження Telegram API).

```python
        send_notification(order)
```

```python
import requests

api_key = 'NGZmYzg4YjY2MTE0OGRkNTBmYWM1NmFjZjE5YWI3MDEwMzc4YmNjZA'
base_url = 'https://keycrm.app/api/v2'

    'url': 'https://sys-plus-bot.onrender.com/webhook/keycrm',
    'events': ['order.created', 'order.status_changed', 'client.created'],
```
